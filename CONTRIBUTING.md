# Contributing to AI Site Connector 🚀

Thank you for your interest in contributing to AI Site Connector! We welcome contributions from everyone.

## Code of Conduct

Please be respectful and professional in all your interactions. We follow the general principles of the [Contributor Covenant](https://www.contributor-covenant.org/).

## How Can I Contribute?

### Reporting Bugs

- Check the [GitHub Issues](https://github.com/jigs10/ai-site-connector/issues) to see if the bug has already been reported.
- If not, create a new issue with a clear title and description, including steps to reproduce the bug.

### Suggesting Features

- Check the [GitHub Issues](https://github.com/jigs10/ai-site-connector/issues) to see if the feature has already been suggested.
- If not, create a new issue and describe the feature you'd like to see.

### Contributing Code

1.  **Fork the repository**.
2.  **Create a new branch** for your feature or bug fix: `git checkout -b my-feature-branch`.
3.  **Make your changes** and ensure they follow the project's style.
4.  **Run tests and build** (see [Development Setup](#development-setup)).
5.  **Commit your changes** with a descriptive message.
6.  **Push your branch** to your fork: `git push origin my-feature-branch`.
7.  **Submit a Pull Request**.

## Development Setup

### Prerequisites

- Node.js (v18 or higher)
- npm

### Setup Steps

1.  **Clone your fork**:
    ```bash
    git clone https://github.com/YOUR_USERNAME/ai-site-connector.git
    cd ai-site-connector
    ```
2.  **Install dependencies**:
    ```bash
    npm install
    ```
3.  **Set up environment variables**:
    Copy `.env.example` to `.env` and fill in your API keys:
    ```bash
    cp .env.example .env
    ```
4.  **Run in development mode**:
    ```bash
    npm run dev -- <command>
    ```
    For example:
    ```bash
    npm run dev -- chat
    ```
5.  **Build the project**:
    ```bash
    npm run build
    ```

## Project Structure

- `src/`: Source code.
  - `bin.ts`: CLI entry point.
  - `index.ts`: Library entry point.
  - `cli/`: CLI commands and prompts.
  - `core/`: Core logic (AI models, scraping, etc.).
- `dist/`: Compiled code (generated by `npm run build`).

## Pull Request Process

- Ensure your code follows the existing style and conventions.
- Keep pull requests small and focused on a single change.
- Update the documentation (if applicable).
- A maintainer will review your pull request as soon as possible.
